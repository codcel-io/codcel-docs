# Contributing to Codcel Documentation

Thank you for your interest in improving the Codcel documentation.

The goal of this repository is to make Codcel easier to understand, learn, and use.  
We welcome contributions that improve clarity, correctness, and completeness.

This repository contains **documentation only**.  
Changes to the Codcel engine or other source code should be submitted to the relevant repository.

---

# Ways to Contribute

You can contribute in several ways:

• Fix typos or unclear explanations  
• Improve examples or tutorials  
• Add missing documentation  
• Clarify behaviour or edge cases  
• Improve structure or navigation

Even small improvements are valuable.

---

# Reporting Issues

If you find a problem with the documentation, please open a **GitHub Issue**.

When reporting an issue, it helps if you include:

• A link to the page or file  
• A description of the problem  
• Suggested wording or correction (if possible)

---

# Submitting Documentation Changes

Documentation improvements should be submitted via **Pull Requests**.

Typical workflow:

1. Fork this repository
2. Create a new branch for your change
3. Make your edits
4. Submit a Pull Request describing the change

Example:
    git checkout -b fix-annotation-docs


Please keep pull requests focused on a single improvement where possible.

---

# Documentation Style Guidelines

When contributing documentation, please follow these guidelines:

### Clarity first
Documentation should prioritize clarity over cleverness.

Prefer simple explanations that are easy to understand.

### Be practical
Whenever possible include:

• examples  
• sample Excel snippets  
• expected outputs  
• common pitfalls

### Keep formatting consistent

Use:

• Markdown headings (`#`, `##`, `###`)
• bullet lists for readability
• code blocks for examples

---

# Product Bugs

If you encountered an issue using the **Codcel application itself**, such as problems converting an Excel workbook to code, please report it via:

https://codcel.io/contact/bugs

or email:

bugs@codcel.io

To help us resolve the issue quickly, include:

• what you were trying to do  
• the error message or logs  
• screenshots if possible  
• environment details (OS, Excel version, Codcel version)  

---

# Feature Requests

Ideas for new capabilities, supported languages, integrations, or workflow improvements can be submitted via:

https://codcel.io/contact/features

or email:

features@codcel.io

Please include:

• a description of the feature  
• the problem it would solve  
• your current workaround (if any)  
• whether it is critical or a nice-to-have

---

# Confidential Data

Excel workbooks and generated code may contain sensitive information.

If you are sharing files:

• remove confidential or regulated data  
• anonymize business information  
• provide minimal examples where possible

Minimal reproducible examples help us diagnose issues faster.
